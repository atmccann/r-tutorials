The plan

Our main goal today is to be comfortable with the R console and its basic data types and syntax, and to use it to ask, answer and visualize journalistically relevant questions. This class will be project-based, since, in my experience, doing actual projects is the best way to learn anything. We might not get to everything today, but do speak up if you have questions or goals that are not covered in the lessons.

Before we start breaking things, let's get our workspace set up properly. Make a folder on your desktop called cuny-tutorial. In that folder, make a new file called tutorials.R in a text editor and save it to your project folder. Start by typing your working directory in your R text file and pasting it into the R console. (Cutting and pasting from static files is not the classiest way to do things, since there's an R IDE now, but this is how the Godfather does it, so it's good enough for us.)

  
    setwd("~/Desktop/cuny-tutorial")

Start typing things into the console and discuss what happens after each.

Questions/thoughts?

R isn't hard, but it can feel difficult because it's picky. The difference between a map or chart that works and one that's broken can sometimes just be a missing argument to a method (like stringsAsFactors=F or horiz=T), which can be frustrating. Rather than try to cover all of thes eexhaustively, we'll just start making some projects and learn as we go. With that, let's go to our first project, visualizing Americans' attitudes about same-sex marriage.
